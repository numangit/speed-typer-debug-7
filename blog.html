Local Storage : The localStorage object stores data with no expiration date and its not deleted when the browser is closed. The Data is available for future session.
Session Storage : The sessionStorage object stores data for only one session and its deleted when the browser is closed. The Data is not available for future session.

Global Scope : Variables with global scope can be accessible from anywhere in the code.
Block Scope : Variables with block scope can only be accessible within that particular block of code it was declared.

The event loop plays a big role in JavaScript's asynchronous programming. Whenever an async function is called, it is sent to a browser API.Based on the command received from the call stack, the API starts its own single-threaded operation.The call stack keeps track of all the operations in line to be executed. When tasks are completed form broswer API it is sent to event queries.The event loop constantly checks whether the call stack is empty. If it is empty, new functions are added from the event queue.

Here are few situations when we get an undefine data type:

  • Variable that is not initialized.(not putting a value for the variable)
  • Trying to get a value from a Function with no return.
  • Parameter that is not passed will be undefined.
  • Trying to get a Return Value when the return value is not given in a function.(just writing Return without mentioning what will be returned will give undefined value)
  • Accessing property that doesn't exist in an object will give you undefined.
  • Accessing array elements outside of the index range. (Trying accessing element index that doesn't exist will give you undefined)
  • Deleting an element inside an array and trying to acces it.
  • if a value is directly set to undefined.
